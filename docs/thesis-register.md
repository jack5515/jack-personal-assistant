# Thesis Register

这份文档用于把反复出现、值得长期指导判断的 thesis 单独收口。

目标不是重复 `MEMORY.md`，而是把那些会反复影响巡检、产品判断、创业判断的核心命题单独列出来，方便：
- 被新 signal 更新
- 被反例挑战
- 反向指导下一轮 watchlist

## Use Rule

只有满足下面至少一条的判断，才适合进入 thesis register：
- 已被多个 signal 反复印证
- 明显影响产品路线、创业方向或资源分配
- 未来 1-4 周内仍值得持续验证

每条 thesis 默认写 6 个字段：
- `Thesis`
- `为什么当前成立`
- `支持信号`
- `削弱/推翻条件`
- `当前状态`
- `下一轮要补什么证据`

## Thesis List

### 1. AI 产品竞争正在从“模型能力”加速转向“记忆迁移 + 工作流承接 + 服务体验”
- 为什么当前成立：头部产品越来越强调历史对话迁移、长期记忆、跨任务承接和工作流，而不只是 benchmark。
- 支持信号：Gemini 的聊天记录/AI memories 迁移；Claw 产品的长期记忆、持续执行、IM 接入成为核心卖点。
- 削弱/推翻条件：如果后续用户选择产品主要还是由单次模型效果和价格决定，而不是由记忆与工作流承接决定。
- 当前状态：进行中，偏强。
- 下一轮要补什么证据：继续盯 `memory import`、`history migration`、`project memory`、`cross-device continuity` 相关产品动作和用户反馈。

### 2. Agent 产品真正值钱的层，不是角色，而是服务配置与交付体验
- 为什么当前成立：底层能力会逐步收敛，真正形成差异的越来越是默认配置、汇报方式、恢复机制、记忆策略、权限与状态可视化。
- 支持信号：`卖服务，不卖角色` 已成为长期判断；Claw 赛道用户越来越在意稳不稳、值不值、出了问题有没有人管。
- 削弱/推翻条件：如果未来用户持续为强人格壳子和 agent identity 付费，而对服务系统差异不敏感。
- 当前状态：高置信度。
- 下一轮要补什么证据：继续看 Kimi Claw / OpenClaw / 其他 Agent 产品是否在加强服务配置层，而不是继续堆角色包装。

### 3. 对 Jack 更有价值的 AI assistant，不是更会回答，而是更像一个认知-决策-执行系统
- 为什么当前成立：对 Jack 这类 AI PM / 创业型用户，核心价值不在单次问答，而在持续情报、判断提纯、执行放大和长期记忆。
- 支持信号：AI 巡检日报、背景知识、双发链路、判断框架、signal-thesis-loop 都在往这个方向演化。
- 削弱/推翻条件：如果这些系统长期没有带来更早判断、更少遗漏、更高质量行动，而只是增加维护成本。
- 当前状态：已建立骨架，待验证效果。
- 下一轮要补什么证据：持续看最近 3 天哪些输出真的影响了 Jack 的判断、行动和节奏，而不只是“送达成功”。
