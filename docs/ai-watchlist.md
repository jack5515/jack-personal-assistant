# AI Watchlist

这份文档定义接下来一轮巡检应该重点盯什么，避免 AI 巡检退回泛扫描。

目标：
- 让巡检更像有方向的雷达
- 让 thesis 反向塑造 signal 搜索范围
- 降低普通新闻占比，提高高价值 signal 命中率

## Current Watchlist

### 1. Memory / Migration / Continuity
重点盯：
- 聊天历史迁移
- memory import/export
- project memory
- cross-device continuity
- 从“聊天产品”升级成“长期工作流承接层”的动作

为什么盯：
- 这条线直接关系到 AI 产品竞争是否进入“记忆资产迁移权”阶段。

### 2. Real-time Live Agent
重点盯：
- 实时语音 + 视觉输入
- 实时打断
- 实时状态更新
- Live API 进入真实产品的落地案例

为什么盯：
- 这条线直接关系到 Agent 交互是否从 turn-based chatbot 升级到更自然的实时协作。

### 3. Claw / Agent Service Layer
重点盯：
- 长期记忆
- 任务恢复
- 自动重试
- 权限可视化
- 审批机制
- 状态可解释性

为什么盯：
- 这条线直接关系到 Agent 产品到底是在做“服务系统”，还是停留在“能力 demo”。

### 4. AI Product Pricing / Packaging
重点盯：
- 订阅分层
- 免费额度变化
- 积分/买断模式
- 记忆、搜索、浏览器、长期执行是否进入收费墙

为什么盯：
- 这条线直接关系到 AI 产品商业化是否在从“模型收费”走向“服务体验收费”。

### 5. PM / Workflow Augmentation
重点盯：
- AI PM 工作流
- 评测系统
- human-in-the-loop
- AI 文档/原型/评审工作流

为什么盯：
- 这条线直接关系到 Jack 本人作为 AI PM 的能力升级和方法论沉淀。

## Exclusions

当前默认降权：
- 普通融资新闻
- 没有明确产品动作的泛观点文章
- 单纯“某模型更强了”但没有产品承接的更新
- 没有公开证据链的社区传闻

## Update Rule

每次强 signal 后，默认问：
- 这条内容要不要进入 watchlist？
- 它会增强哪条 thesis？
- 它会不会替换掉当前某条低价值 watch item？
