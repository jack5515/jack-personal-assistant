# Signal Thesis Loop

这份文档定义 `巡检日报 -> signal -> insight -> thesis -> 长期记忆 -> 下一轮巡检` 的闭环。

目标：
- 不让巡检只停留在资讯层
- 不让长期记忆只停留在结论仓库
- 让外部 signal 和内部判断持续互相修正

## Loop

1. `巡检`
- 从官方源、社区、竞品动作里抓 signal
- 只保留高价值内容

2. `insight`
- 判断为什么这不是普通新闻
- 提炼对产品、创业或 PM 判断的意义

3. `thesis`
- 再往上一层，写出更长期的模式判断
- 例如：竞争焦点是不是正在迁移？用户心智是不是在改变？某种产品范式是不是正在成形？

4. `长期记忆`
- 只有值得复用的 thesis、框架和产品判断，才进入 `MEMORY.md` 或正式文档
- 不是每条巡检都直接沉淀

5. `下一轮巡检`
- 从 thesis 反推新的 watchlist
- 下一轮优先盯哪些公司、哪些功能、哪些用户反馈、哪些验证信号

## Practical Rule

每次强 signal，默认问 3 个问题：

1. 这条 signal 说明什么更长期的变化？
2. 它会不会改变当前某条 thesis？
3. 下一轮巡检该继续盯什么来验证或推翻它？

## Failure Modes

最常见的失败方式有 3 种：

- `只有新闻，没有 insight`
- `只有 insight，没有 thesis`
- `有 thesis，但没有回流到下一轮 watchlist`

## Minimal Upgrade Path

如果不想一步做太重，最小升级也至少要做到：

- 巡检简报里多一栏 `Thesis 提炼`
- 对特别强的 signal，补一句 `下一轮巡检要盯什么`
- 每隔几天回看一次，把重复出现的 thesis 提炼进长期文档
