# AI Trend Watch Task

目标：自动巡检 X 与小红书上关于 AI 产品、AI 模型的有趣信息；采用事件驱动推送，而不是固定时间简报。只有出现真正值得 Jack 关注的高价值信号时，才立即双发到 Jack 的飞书私聊和微信。

## 主 agent / 子 agent
- 主 agent：主会话 agent（负责统筹、判断、汇报）
- 子 agent：`product-trend-researcher`（负责发现和提炼）
- 校验子 agent：`testing-reality-checker`（必要时做交叉校验）

## 关注范围
优先关注以下主题：
- AI 产品新功能、新入口、新交互、新工作流
- AI 模型发布、升级、价格变化、上下文变化、速度/能力变化
- ChatGPT、Claude、Gemini、Copilot、Kimi、Manus、Claw/OpenClaw 相关动态
- Agent、自动化、workflow、memory、browser use、tool use 相关动态
- 在 X / 小红书上高密度出现的真实用户反馈、口碑变化、吐槽点、惊喜点
- 对 Jack 有产品启发的包装、增长、定价、留存、用户教育动作

## 巡检策略
- 默认优先检查官方源，再看媒体或公开讨论，最后才看 X / 小红书等社交平台
- 官方源白名单优先包括：`OpenAI`、`Anthropic`、`Google/Gemini`、`Microsoft/Copilot`、`GitHub`、`Perplexity`、`Qwen/通义`、`Kimi/月之暗面`、`OpenClaw`
- `X / 小红书` 默认只作为发现线索或用户反馈补充，不作为唯一证据
- 每次巡检优先看最近 `3` 天；若强信号不足，再扩到最近 `7` 天
- 每次最多发 `2-3` 条；宁缺毋滥
- 每次输出里至少 `1` 条必须带官方链接，否则宁可不发

## 价值阈值
只有满足以下任一条件，才值得发给 Jack：
1. 有一个高强度 signal：明显的新产品动作 / 新模型发布 / 重大能力变化 / 明确价格变化 / 高价值用户反馈
2. 同一主题在多个来源上形成交叉印证
3. 能直接提炼出对 AI 产品判断有帮助的启发，而不是普通新闻转述

如果没有达到阈值，回复必须是：NO_REPLY

## 输出要求
- 用中文
- 只发 1-3 条最值得看的 signal / insight，不要刷屏
- 最终命中的 `AI 巡检简报` 默认走飞书 + 微信双发，不再只发单通道
- 默认优先产出 `insight`，不是普通新闻摘要；如果只有新闻、没有 insight，降级为 `候选` 或不发
- 结果默认区分：`已完成 / 候选 / 低置信度 / 待验证`
- 对小红书、X、Reddit 或其他社区上无法充分证实的信息，必须明确标低置信度或待验证
- 不要把猜测包装成确定事实

## 默认执行方案
- 第一层：优先用 `step web_search` 做硬信号初筛
- 第二层：如果搜索结果偏泛、像普通新闻摘要、时间窗不准，必须 fallback 到 `Codex`
- 第三层：`Codex` 优先抓 `GitHub issues/discussions`、`Reddit threads`、`X posts` 里的真实社区 signal
- 最终只保留有产品 insight 的内容；若无足够 insight，允许明确写 `暂无强 insight`
- 默认优先级：`GitHub > Reddit > X > 其他社区`
- 默认至少保留 `1` 个可点击证据链接；高质量版本尽量每条 insight 都带具体链接

## 来源约束
- 优先使用公开可访问网页、web search 结果、可抓取页面
- 可以把 X 和小红书作为线索源，但当证据不足时必须标注清楚
- 若某条内容无法找到公开链接，宁可降级为 `低置信度` 或不发

## 风格
- 简洁
- 直接
- 有判断
- 不废话
- 不为了凑数而发消息
